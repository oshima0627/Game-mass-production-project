# Releases 台帳

実装フェーズに入った作品の一覧。**このファイルが作品ステータスの唯一の正本**。各作品の設計書 `design/<game-slug>.md` にはステータスを書かない（ドリフト防止）。

## 日付フォーマット

すべて ISO 8601（`YYYY-MM-DD`）を使用。未定は空欄、未達は `-`。

## ステータスの取りうる値

| ステータス | 意味 |
|------|------|
| `design` | 設計フェーズ（本リポジトリの `design/<game-slug>.md` で作業中） |
| `in-development` | 実装リポジトリ作成済み、開発中 |
| `soft-launched` | 特定国でソフトローンチ開始、指標検証中 |
| `global` | グローバルリリース済み |
| `maintenance` | 運営フェーズ（バグ修正・イベント運用中心） |
| `sunset` | サービス終了 |

## 更新タイミング

- `in-development`: 実装リポジトリ作成直後（STEP 4 完了時）
- `soft-launched`: フィリピン/カナダ等でのストア公開時
- `global`: 全地域公開時
- `maintenance`: アクティブ開発終了時
- `sunset`: サービス終了決定時

ステータス遷移のタイミングで作品固有の学びを `design/knowledge/` に抽出する。
