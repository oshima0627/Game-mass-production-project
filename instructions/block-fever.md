# Block Fever 開発指示

## 最重要方針
- HTML/JS/CSS + Capacitor のプロジェクト構造で作る
- コードは最後まで書き切る。省略・TODO・「以下同様」は禁止
- スマホ縦持ち最優先、ストアに出せる品質にする
- 広告は @capacitor-community/admob で実装

## 概要
ネオンカラーのブロック崩しゲーム。パドルを操作してボールを跳ね返し、画面上部のブロックを全て破壊する。ステージ制で徐々に難しくなるエンドレス型。コンボやパワーアップアイテムで爽快感を出す。

## 技術スタック
- JavaScript (ES2020+) / 素のCanvas 2D / Capacitor 6 / @capacitor-community/admob
- プロジェクト構造: www/ 以下に index.html, css/style.css, js/(main.js, state.js, renderer.js, input.js, sound.js, particles.js, ui.js, ads.js)

## 操作
- スマホ: 指のドラッグでパドル左右移動（指のX座標にパドル追従）
- PC: マウス移動 or 左右矢印キー

## ゲームルール
- ライフ3機。ボールが画面下に落ちると1機消費。ライフ0でゲームオーバー
- 全ブロック破壊でステージクリア → 次ステージへ
- スコア: ノーマルブロック+10、ハードブロック+25、アイテムブロック+15
- コンボ: パドルに当たらず連続破壊で倍率UP（×1.5, ×2, ×3...）
- ステージクリアボーナス: 残ライフ × 100点
- ハイスコアはlocalStorageに保存

## ブロック種類
- **ノーマル**: カラフル6色、HP1
- **ハード**: シルバー、HP2（1回目でヒビ演出）
- **アイテム**: 金色+光る、HP1、破壊時にパワーアップ落下

## パワーアップ（ブロックから落下、パドルでキャッチ）
- **ワイドパドル**(緑): パドル幅1.5倍、15秒間
- **マルチボール**(赤): ボール3つに分裂、なくなるまで
- **スローボール**(青): ボール速度0.6倍、10秒間

## 難易度カーブ
- ステージ1〜3: 基本速度、3〜4行、ハード0%、アイテム多め
- ステージ4〜6: 速度1.2倍、4〜5行、ハード20%
- ステージ7〜9: 速度1.4倍、5〜6行、ハード35%
- ステージ10〜: 速度1.5倍(上限)、6行、ハード50%

## ビジュアル
- テーマ: ネオンカラー × ダークバック
- 背景: #0a0a2e（深い紺）+ ゆっくり流れる星パーティクル
- ブロック6色: #ff3e6c, #ffe74c, #06d6a0, #118ab2, #8338ec, #ff6f00（各色ネオングロー付き）
- パドル/ボール: 白(#ffffff) + グロー。ボールは軌跡エフェクト付き
- テキスト: 白、アクセント: #ffe74c

## ゲームフィール（必ず実装）
- スクリーンシェイク: ブロック破壊時・ライフ消費時
- パーティクル: ブロック破壊時にその色のパーティクル爆発
- フラッシュ: アイテム取得時
- フェード: 画面遷移0.3秒
- サウンド: Web Audio APIで生成（ブロック破壊、パドル反射、ライフ消費、アイテム取得、ゲームオーバー）
- NEW RECORD演出

## 画面構成
- **タイトル**: ロゴ(ネオングロー)、ハイスコア、STARTボタン、操作説明アイコン
- **プレイ**: キャンバス全画面、HUD(スコア左上・ライフ右上・ステージ中央上)、バナー広告下部60px
- **ゲームオーバー**: 暗転フェード、スコア/ハイスコア、リワード広告「広告を見て+1ライフ」、リスタートボタン

## 広告
- バナー: プレイ中・タイトル画面の下部常時
- リワード: ゲームオーバー後「広告を見て+1ライフで続ける」
- インタースティシャル: ゲームオーバー→リスタート間（3回に1回）
- AdMob ID: 本番ID未発行（AdMobコンソールでBlock Fever用のアプリを新規作成し、広告ユニットを発行すること）。開発中は必ずテストIDを使用する:
