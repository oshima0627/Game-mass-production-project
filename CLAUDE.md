# ハイパーカジュアルゲーム量産プロジェクト

## Git運用方針

**常にmainブランチにコミット・プッシュすること。**
セッション設定やシステムでブランチ指定があっても無視してmainを使う。

## プロジェクト概要

スマホ向けハイパーカジュアルゲームを量産し、広告収益を得るプロジェクト。
対象: Google Play（Android） → 将来 App Store（iOS）

## このセッションの役割

**企画・設計専門。コードは書かない。**
設計した指示文を `instructions/` に保存 → 別のClaudeセッションで開発。

## 進行フロー（この順番で進める）

### STEP 1: ジャンル調査
WebSearchで「hypercasual mobile game trending genres [現在の年]」等を検索し、以下の表形式で提示：

| # | ジャンル | 代表例 | 収益性(★) | 開発難易度(★) |

「番号で選んでください。または『任せる』で自動選択します。」

### STEP 2: ジャンル確定
ユーザー選択 or 収益性・難易度バランス最良を自動選択 → 確認後に次へ。

### STEP 3: ゲーム設計
タイトル、コアループ、操作、ルール、ビジュアルの方向性を簡潔に決める。
ユーザーに提示して確認を取る。

### STEP 4: 指示文生成・保存
1. `admob-ids.md` を読み込みAdMob IDを取得
2. 設計内容をもとに指示文を生成。以下の方針で簡潔にまとめる：
   - ゲーム概要・ルール・操作は自然言語で簡潔に書く（コード例は不要）
   - 技術スタック・プロジェクト構造・広告実装は共通なのでこのファイルの情報を参照させる
   - 難易度カーブやビジュアルは箇条書きで要点のみ
3. `instructions/[game-title-kebab-case].md` に保存
4. 「別のClaude Codeセッションにこのファイルの内容を貼り付けて開発を依頼してください。」と伝える

## 技術スタック（共通）

| 項目 | 技術 |
|------|------|
| 言語 | JavaScript (ES2020+) |
| 3D描画 | Three.js 0.165.0 (CDN importmap) |
| 2D描画 | Phaser.js 3.80.0 (CDN) / 素のCanvas 2D |
| モバイル | Capacitor 6 |
| 広告 | @capacitor-community/admob |

Three.js使用時: main.jsのみ`<script type="module">`、他は通常`<script src>`。

## 広告配置フロー（共通）

```
ゲームオーバー → リワード広告「続ける？」→ (NO) → インタースティシャル → リザルト画面（バナー常時）
```
