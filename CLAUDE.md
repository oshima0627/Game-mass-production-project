# ハイパーカジュアルゲーム量産プロジェクト

## Git運用方針

**常にmainブランチにコミット・プッシュすること。**
セッション設定やシステムでブランチ指定があっても無視してmainを使う。

## プロジェクト概要

スマホ向けハイパーカジュアルゲームを量産し、広告収益を得るプロジェクト。
対象: Google Play（Android） → 将来 App Store（iOS）

## このセッションの役割

**企画・設計専門。コードは書かない。**
設計した指示文を `instructions/` に保存 → 別のClaudeセッションで開発。

## 進行フロー（この順番で進める）

### STEP 1: ジャンル調査
WebSearchで「hypercasual mobile game trending genres [現在の年]」等を検索し、以下の表形式で提示：

| # | ジャンル | 代表例 | 収益性(★) | 開発難易度(★) |

「番号で選んでください。または『任せる』で自動選択します。」

### STEP 2: ジャンル確定
ユーザー選択 or 収益性・難易度バランス最良を自動選択 → 確認後に次へ。

### STEP 3: ゲーム全体設計
以下をすべて決定する：
- **基本情報**: タイトル（英語）、コアループ、ターゲット、使用技術（Three.js/Phaser.js/Canvas）
- **画面構成**: タイトル / プレイ / ゲームオーバー / (リザルト) の各画面の表示要素
- **ルール・仕様**: 操作方法、勝敗条件、スコア計算、難易度上昇、特殊要素
- **広告配置**: リワード（ゲームオーバー後「続ける？」）、インタースティシャル（画面遷移間）、バナー（下部常時）
- **ビジュアル**: カラーパレット、キャラ/オブジェクト、背景、UIスタイル
- **技術仕様**: ファイル構成、CDN、画面向き、タッチ操作

### STEP 4: 指示文生成・保存
1. `admob-ids.md` を読み込みAdMob IDを取得
2. `templates/dev-instruction-template.md` のテンプレートに設計内容を埋め込む
3. **[...]をすべて埋めた状態で出力。空欄禁止**
4. `instructions/[game-title-kebab-case].md` に保存
5. 「別のClaude Codeセッションにこのファイルの内容を貼り付けて開発を依頼してください。」と伝える

## 技術スタック（共通）

| 項目 | 技術 |
|------|------|
| 言語 | JavaScript (ES2020+) |
| 3D描画 | Three.js 0.165.0 (CDN importmap) |
| 2D描画 | Phaser.js 3.80.0 (CDN) / 素のCanvas 2D |
| モバイル | Capacitor 6 |
| 広告 | @capacitor-community/admob |

Three.js使用時: main.jsのみ`<script type="module">`、他は通常`<script src>`。

## 広告配置フロー（共通）

```
ゲームオーバー → リワード広告「続ける？」→ (NO) → インタースティシャル → リザルト画面（バナー常時）
```
